🎯 Problem & Vision

Cafés deserve software as
premium as their coffee

Most POS tools are bloated, slow, and cloud-locked. BrewMatic proves you can have enterprise-grade intelligence with zero cloud lock-in.

🌐

Cloud Downtime = Lost Sales

When the internet drops, many POS systems freeze entirely. BrewMatic transparently falls back to localStorage so the show never stops.

🖨️

Disconnected Systems

Orders hand-written on paper, kitchen staff squinting at screens, admin guessing from receipts. BrewMatic unifies everything in real-time.

🤖

No Intelligent Upselling

No suggestions, no smart promotions, no automated discounts. BrewMatic ships a live co-occurrence AI engine and automated promo rules — out of the box.

📊

Insight Blind Spots

Who is your best-selling product at 6 PM on weekdays? BrewMatic's filterable dashboard answers these in real-time with charts you can export.

🤖 Smart AI Engine

Live product suggestions,
powered by your own data

backend/routes/ai.js
// Co-occurrence recommender from paid orders
router.get('/suggestions', (req, res) => {
  const cart = req.query.cart.split(',')

  // Build co-occurrence from DB
  const rows = db.prepare(`
    SELECT oi.orderId, oi.productId
    FROM order_items oi
    JOIN orders o ON o.id=oi.orderId
    WHERE o.status='paid'`).all()

  // Score & rank by co-occurrence
  const scored = rankByCoOccurrence(rows, cart)

  // Cold-start: top sellers in same category
  if (scored.length < 3) fallbackTopSellers(cart)

  res.json(ranked.slice(0, 3))
})
✨

Live Co-Occurrence Model

Every time an order is paid, the model grows smarter. Products frequently bought together get ranked higher — no ML library needed, pure SQL.

🔥

Cold-Start Fallback

For new cafés with no history, the engine falls back to top-selling products in the same category — always showing something relevant.

⚡

Zero Latency UX

Suggestions are fetched with a 250ms debounce as the cart changes. Results appear inline in the cart — one tap to add. Items already in cart are filtered out.

🎯

Automated Promotions

Beyond suggestions — rule-based promos fire automatically: "Coffee Lovers" (3× Cappuccino → 15% off) and "Big Order Bonus" (₹800+ → ₹100 flat off) apply without cashier input.

🔒 Built to Last

Offline-first.
Live-sync second.

📴

Offline Fallback

If the backend goes down during a demo, the frontend store transparently falls back to localStorage. Orders keep flowing, receipts still print. When the backend comes back up, state is reconciled.

⚡

WebSocket Live Sync

Every state change (new order, kitchen advance, payment) is broadcast over WebSocket to every connected device. The Kitchen Display updates the moment the cashier taps "Send to Kitchen" — no refresh, zero delay.

🔑

Secrets-Safe Architecture

Passwords, UPI ID — nothing is hardcoded. All sensitive values live in .env which is gitignored. The .env.example template ships safely in the repo.

🌱

Single Source of Truth

src/store/seed.js is the one module that defines all demo data. Both the Vite frontend and the Node backend seeder import it directly — zero duplication, zero drift.
